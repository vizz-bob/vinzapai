================================================================================
                      QUICK START COMMANDS
================================================================================

DEVELOPMENT:

Backend:
  cd vinzapai-backend
  python -m venv venv
  venv\Scripts\activate  (Windows) or source venv/bin/activate (Unix)
  pip install -r requirements.txt
  python manage.py migrate
  python manage.py runserver

Frontend:
  cd vinzapai-frontend
  npm install
  npm start

Access Dashboard: http://localhost:3000
Access Admin:     http://localhost:8000/admin/
Default Login:    admin / vinzap123

BUILD FOR PRODUCTION:

  cd vinzapai-frontend
  npm run build
  
  xcopy build ..\vinzapai-backend\frontend_build /E /I /Y
  
  cd ..\vinzapai-backend
  python manage.py collectstatic --noinput --settings=backend.settings_standalone
  
  pyinstaller -y vinzapai_backend.spec
  
  "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" ..\installer.iss

Output: VinzapAI_Setup.exe (ready to distribute)

================================================================================
                    NEXT STEPS FOR DEPLOYMENT
================================================================================

1. Initialize Git Repository:
   git init
   git add .
   git commit -m "Initial commit: Complete VinzapAI SaaS dashboard"
   git remote add origin https://github.com/yourusername/vinzapai.git
   git push -u origin main

2. Configure GitHub Secrets (for CI/CD):
   - No secrets required, builds automatically on push

3. Setup GitHub Releases:
   - GitHub Actions automatically creates installer artifacts
   - Download from Actions tab

4. Install Dependencies:
   cd vinzapai-backend && pip install -r requirements.txt
   cd ../vinzapai-frontend && npm install

5. Run Migrations:
   python manage.py migrate

6. Collect Static Files:
   python manage.py collectstatic

7. Create Superuser:
   python manage.py createsuperuser

8. Start Development:
   python manage.py runserver (backend)
   npm start (frontend in another terminal)

9. Build for Distribution:
   Follow build commands in Quick Start section

================================================================================
                         CUSTOMIZATION NOTES
================================================================================

The project is fully customizable:

- Theme Colors: Edit src/index.css for color variables
- Data Models: Add fields to models.py in any app
- API Endpoints: Create new ViewSets and register in urls.py
- Pages: Create new components in src/pages/
- Navigation: Edit Sidebar.jsx to add/remove menu items
- Admin Interface: Customize admin.py for each app
- Charts: Use any Recharts component in Analytics

All changes are automatically reflected in both development and production builds.
